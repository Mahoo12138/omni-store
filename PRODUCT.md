# Product

## Register

product

## Users

自部署用户：个人、家庭和小团队的管理员与成员。他们在自己的服务器/NAS 上运行 OmniStore，通过浏览器管理本地目录中的真实文件；访客通过公开链接浏览和下载被共享的目录。使用场景是日常工具型操作：找文件、传文件、拿图床外链，通常在桌面浏览器，移动端需可用。

## Product Purpose

OmniStore 是以本地目录为真实数据源的轻量自部署存储中心：公开网盘、私有网页文件管理器、WebDAV、图床（登录用户 + 可选匿名）。成功标准不是功能多，而是每条核心路径稳定、安全、可解释（README §29）。

## Brand Personality

清爽、可靠、克制。现代管理后台风格（README §2.3）：界面为任务服务，不抢戏。工具应当消失在任务里。

## Anti-references

- 企业级控制台的复杂度（多层抽屉、无尽配置页）。
- 营销化的 SaaS 首页语言（大数字指标卡、渐变文字）。
- 伪造数据的装饰性面板（没有后端支撑的统计、活动流不做假数据）。

## Design Principles

1. **设计图即契约**：docs/ 下的 UI 设计稿定义视觉语言（蓝主色、白卡片、浅灰底、彩色图标砖），实现忠于稿件的气质而非像素复刻；没有后端数据支撑的元素（配额、全局搜索）宁可删除不可造假。
2. **同一套词汇表**：公开侧与登录侧共用同一文件表格、同一按钮、同一图标风格；同一动作在任何屏幕长得一样。
3. **状态完整**：每个交互组件有 hover / focus / disabled / loading / empty / error；空状态教用户下一步做什么。
4. **密度为效率服务**：文件表格可以密，操作路径必须短；破坏性操作（删除）必须有明确确认文案。
5. **移动端可用而非惊艳**：侧栏折叠、表格可横滚，公开网盘/登录/图床/基础文件操作在手机上完整可用（README §24.4）。

## Accessibility & Inclusion

WCAG 2.1 AA 为目标：正文对比度 ≥4.5:1，焦点可见，键盘可达；动效尊重 prefers-reduced-motion。界面语言为简体中文。
