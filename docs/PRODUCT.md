# 产品定义

本文档说明 OmniStore 面向谁、解决什么问题，以及 MVP 的明确边界。

## 目标用户

OmniStore 面向在自有服务器或 NAS 上管理文件的个人、家庭与小团队。核心角色包括负责配置存储源和权限的管理员、进行日常上传与整理的成员，以及通过公开链接浏览和下载内容的访客。主要使用环境是桌面浏览器，移动端必须覆盖登录、公开网盘、图床和基础文件操作。

## 产品目标

OmniStore 是以本地目录为真实数据源的轻量自部署存储中心，统一提供公开网盘、私有网页文件管理器、WebDAV 与图床。它的价值不在于堆叠功能，而在于让用户清楚知道文件在哪里、当前能做什么、操作会产生什么结果。

成功标准是核心路径稳定、安全、可解释：配置存储源后即可管理真实文件；公开与私有访问边界明确；匿名能力仅在用户主动启用时出现；任何没有后端数据支撑的信息都不伪造。

## 品牌个性

清爽、可靠、克制。OmniStore 应当像一只“安静的文件柜”：结构清楚、拿取顺手、长期使用不疲劳。界面为任务服务，不追求戏剧化表现；工具应当尽量消失在任务里。

## 设计反例

- 企业级控制台的复杂度：多层抽屉、无尽配置页、为了显得专业而增加的层级。
- 营销化的 SaaS 首页语言：大数字指标卡、渐变文字、口号式模块。
- 伪造数据的装饰性面板：没有后端支撑的配额、趋势、活动流或全局统计。

## 设计原则

1. **安静的文件柜**：信息按任务自然归位，常用动作伸手可得，低频设置不抢占注意力。
2. **设计图即契约**：`docs/` 中的界面稿定义整体气质；实现忠于清爽、可靠、克制的体验，不为像素复刻制造虚假能力。
3. **同一套词汇表**：公开侧与登录侧共享按钮、输入框、文件表格、图标和状态语言；同一动作在不同页面保持相同含义与反馈。
4. **状态完整**：交互组件具备 hover、focus-visible、disabled、loading、empty 与 error；空状态说明原因，并给出唯一清晰的下一步。
5. **密度为效率服务**：文件列表可以紧凑，操作路径必须短；危险动作使用明确动词和二次确认，不能靠颜色独自表达风险。
6. **移动端完整可用**：侧栏转为横向导航，表格允许横向滚动；不隐藏完成核心任务所需的功能。

## 无障碍与包容性

目标为 WCAG 2.1 AA：正文对比度不低于 4.5:1，键盘可达，焦点清晰可见，语义状态不只依赖颜色；动效尊重 `prefers-reduced-motion`。产品界面以简体中文为主，文案直接、具体、避免内部技术术语。

## 产品定位

OmniStore 是一款以本地存储源为中心的轻量级自部署存储中心系统。它将本地目录作为核心存储资源，在此基础上提供公开网盘、私有网盘、WebDAV、图床和后续 S3 网关能力。

核心定位：

1. 面向个人、家庭、小团队自部署。
2. 优先轻量、可靠、清晰，而不是企业级复杂能力。
3. 单二进制交付，前端通过 Go `go:embed` 嵌入后端。
4. 默认单实例运行，不支持多副本横向扩展。
5. 本地文件系统是真实数据源，MVP 不维护完整虚拟文件树。
6. SQLite 只保存用户、权限、配置、Session、Token、图床流水、审计日志等系统数据。
7. 所有真实文件操作都必须严格经过权限、路径安全、排除规则和锁检查。

## MVP 必做

MVP 只实现第一条核心闭环：

```text
本地存储源管理
+ 用户登录和权限
+ 公开网盘首页
+ 私有网页文件管理器
+ WebDAV 基础读写
+ 登录用户图床
+ 匿名公共图床
+ 轻量审计日志
```

MVP 必做能力：

1. Linux 生产环境支持，amd64 / arm64。
2. Docker Compose 推荐部署，单二进制直跑作为备用。
3. YAML 配置文件 + 环境变量覆盖。
4. 初始化第一个超级管理员。
5. 超级管理员管理用户、存储源、权限、公开挂载、图床开关。
6. 普通用户使用被分配的存储源。
7. 匿名访客浏览公开网盘和访问公开图床图片。
8. 存储源级权限：只读 / 读写。
9. 存储源全局排除路径规则。
10. 公开网盘虚拟挂载路径。
11. 私有网盘基础文件操作：浏览、上传、下载、删除、重命名、移动、新建目录。
12. WebDAV 基础方法：`OPTIONS / PROPFIND / GET / HEAD / PUT / MKCOL / DELETE / MOVE`。
13. WebDAV 使用用户名 + 独立 WebDAV Token 的 Basic Auth。
14. 登录用户图床，支持网页上传、历史墙、PicGo 兼容上传接口。
15. 匿名公共图床，默认关闭，开启后只允许上传图片。
16. 图床公开 URL 使用 `/i/{image_id}.{ext}`。
17. 请求级内存读写锁。
18. 轻量审计日志。
19. 统一 API 响应和错误格式。
20. 基础响应式前端。
21. 管理员命令行紧急重置密码。

## MVP 明确不做

MVP 不实现以下能力：

1. S3 协议服务。
2. S3 专用端口监听。
3. S3 Multipart。
4. 文件复制。
5. 跨存储源移动。
6. WebDAV `COPY`。
7. WebDAV `LOCK / UNLOCK` 持久锁。
8. WebDAV `PROPPATCH / REPORT / SEARCH / ACL`。
9. 子路径用户权限、目录级 ACL、文件级 ACL。
10. 用户组、角色组、复杂 RBAC。
11. 文件分享链接。
12. 回收站。
13. 文件版本历史。
14. 文件全文搜索。
15. 文件索引数据库。
16. 用户总配额和存储源总配额。
17. 后端缩略图生成和缩略图缓存。
18. 大文件分片上传、断点续传。
19. 在线预览 Office / PDF / 视频转码。
20. PWA 离线能力。
21. Web UI 修改基础设施配置。
22. CORS 跨域部署。
23. 多实例 / K8s 多副本部署。
24. 内置备份恢复。
25. 邮箱找回密码。
26. OAuth / OIDC / LDAP / Passkey。
